# Модуль машинного зрения для контроля дефектов рам (Демонстрационная версия)

Данный микросервис представляет собой демонстрационную версию (MVP) системы автоматизированного технического контроля (ОТК) для завода. Сервис имитирует обработку изображений с камер конвейера, классифицирует типы брака и передает результаты в центральную систему управления.

Проект разработан в рамках производственной практики.

## Взаимодействие в рамках киберфизического комплекса
1. **Нижний уровень**: Промышленный датчик положения фиксирует раму на конвейере, после чего камера делает снимок.
2. **Уровень Edge AI (Данный сервис)**: Python-скрипт обрабатывает изображение с помощью базовой модели YOLOv8. В демонстрационном режиме запущена непрерывная симуляция конвейера с генерацией случайных технологических дефектов.
3. **Промышленная шина данных**: Результаты анализа с указанием ID рамы и коэффициента уверенности сети мгновенно отправляются через брокер сообщений Apache Kafka.
4. **Верхний уровень**: Данные принимает центральный бэкенд-сервис для последующей визуализации на дашборде оператора.

Интеграция со смежным микросервисом: [Репозиторий сервиса контроля качества](https://github.com/Diana-debug-ship-it/Defect-Control-System-Demo)

---

## Функционал демо-версии
* **Режим непрерывного конвейера**: Работа в бесконечном цикле с настраиваемой задержкой между поступающими деталями.
* **Имитация работы ИИ**: Случайная генерация дефектов с плавающим коэффициентом уверенности модели (Confidence Score) в диапазоне от 75% до 98% для проверки отказоустойчивости системы.
* **Моделируемые типы дефектов**:
  * `CRACK` — трещина металла.
  * `GEOMETRY_DISTORTION` — нарушение геометрии лонжеронов.
  * `WELD_DEFECT` — дефект сварочного шва.
  * `MISSING_HOLE` — отсутствие технологического отверстия.
  * `NONE` — дефекты отсутствуют (норма).

## Требования и зависимости
Для работы сервиса необходим Python версии 3.10 или выше.

Установка библиотек:
```bash
pip install ultralytics kafka-python
```

---

## Структура тестовых данных
Перед запуском скрипта необходимо убедиться в наличии директории для тестовых изображений:
```text
cv-service/
├── images/
│   ├── frame_good.png   # Образец качественной детали (Норма)
│   └── frame_bad.jpg    # Образец детали с дефектом поверхности
└── cv-service.py        # Исполняемый файл сервиса
```

---

## Запуск сервиса

1. Убедитесь, что брокер сообщений Apache Kafka активен по адресу `localhost:9092`.
2. Запустите скрипт через терминал:
```bash
python cv-service.py
```

### Остановка симуляции
Для безопасной остановки бесконечного цикла конвейера используйте комбинацию клавиш **`Ctrl + C`** в окне терминала. Скрипт перехватит прерывание `KeyboardInterrupt` и корректно завершит свою работу.
